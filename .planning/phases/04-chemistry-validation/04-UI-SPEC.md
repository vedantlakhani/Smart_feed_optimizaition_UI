## Interaction Contract

### Progressive Disclosure — Three-Tier (CHEM-02)

**Applies to:** r_naoh computation and pH blending method inside `PhaseDetail` in `phase-details-tab.tsx`.

**Tier 1 — Always visible (no interaction):**
Plain English sentence. Static `<p>` tag. Never hidden.
- r_naoh example: "Added {X}% neutralizer to keep pH in the safe operating range."
- pH example: "pH was computed from the hydrogen-ion concentrations of each stream, not from a simple average."

**Tier 2 — Formula (single click to expand):**
Accordion trigger labeled "Show formula" (`text-xs text-slate-400 hover:text-slate-600`).
Formula displayed in `font-data text-xs` inside AccordionContent.
Trigger changes to "Hide formula" when open (Accordion handles this via `data-open` state).

**Tier 3 — Derivation (nested second click to expand):**
Nested `<Accordion multiple>` inside Tier 2 AccordionContent.
Trigger labeled "Show derivation" (`text-xs text-slate-400`).
Derivation text: prose paragraph, `text-xs text-slate-500`, `leading-relaxed`.

**Two-tier only (plain + formula, no derivation):** r_water and r_diesel — simpler formulas do not warrant a derivation tier.

**Accordion API (critical):** Use `multiple` boolean prop, NOT `type="multiple"`. Source: `expert-overrides.tsx` line 270, `accordion.tsx` wraps `@base-ui/react`.

### AssumptionsPanel — Always Visible

AssumptionsPanel renders unconditionally in the "details" tab — no `result` guard, no `showTechnical` gate. It contains only static content. Placement: after `<ExpertOverrides />` in the details tab JSX.

**K-Values section interaction:** No interaction. Each K-value renders as a static row with:
- Plain label text (left)
- Amber badge "Theoretical estimate — not yet calibrated" (right-aligned)
- Plain-English description below the label
- Monospaced value + unit below the description

**MVP Assumptions section interaction:** Accordion with `multiple` prop. Each assumption is an AccordionItem. Trigger: assumption ID + short title. Content: single plain-English sentence. Default state: all collapsed.

### SensitivityNote — Static Inline

No interaction. Renders as a single `<p>` tag directly below cost savings figures. Always visible. Not a toggle, not an accordion. Small and unobtrusive.

## Phase-Specific Visual Rules

1. **SensitivityNote placement is mandatory in three locations:** `impact-header.tsx` (below savings_pct KPI value), `cost-story.tsx` (below savings row in cost table), `cost-story.tsx` (below "Total Cost Reduction" progress bar in Climate Impact section). Missing any one of these three fails CHEM-03.

2. **AssumptionsPanel must NOT be wrapped in `{showTechnical && ...}`** — it renders unconditionally, unlike ExpertOverrides which respects the Plant Manager toggle.

3. **No new accent (ax-cyan) usage is introduced** — cyan remains reserved for optimized/success metrics as established in Phases 1–3.

4. **Amber is single-purpose:** Amber color family (`amber-50`, `amber-300`, `amber-600`, `amber-700`) appears only on the confidence badge. Do not reuse amber on SensitivityNote (use `text-amber-600/80` for the note text per RESEARCH.md pattern).

5. **AssumptionsPanel card shadow:** Use `shadow-card` token (`0 1px 3px 0 rgb(0 0 0 / 0.07)`) matching all other dashboard cards. Do not use `shadow-card-hover` on hover — this is a static information panel, not an interactive card.

6. **Tier 2/3 triggers use subdued text:** `text-xs text-slate-400` for closed state, `text-xs text-slate-600` for open state. This keeps disclosure affordance visible without competing with primary content.
