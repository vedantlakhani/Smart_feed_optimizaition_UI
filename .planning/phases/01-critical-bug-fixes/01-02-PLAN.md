<objective>
Fix two dashboard-side bugs:

BUG-03: `phase-details-tab.tsx:85` computes `btuEff` using hardcoded `18300 * 0.89` instead of `cfg.BTU_diesel * cfg.eta`. `expert-overrides.tsx:PhaseDetailRow` has the same hardcoded values and also hardcodes `W_MIN = 0.5`, `solidEff <= 15`, `saltEff <= 5000`. Fix: use the `cfg: SystemConfig` from `result.config` threaded as a prop through the component hierarchy.

BUG-04: When `result.optimized` is null (no feasible blend), `recipe-tab.tsx` currently shows the generic "No Feed Recipe Yet" empty state — indistinguishable from "not run yet". Fix: split the null guard into two branches so that `result && !result.optimized` shows a distinct "No Feasible Blend Found" panel with the baseline cost.

Purpose: Safety check numbers that ignore user config are misleading for plant operators. A crash-or-silence on infeasible results destroys trust in the product.
Output: Patched phase-details-tab.tsx, expert-overrides.tsx, and recipe-tab.tsx. Dashboard build passes.
</objective>
